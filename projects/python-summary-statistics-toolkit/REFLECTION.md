# Project Reflection

## Summary

This project turned repeated summary calculations into reusable Python functions. The main focus was learning how to use parameters, return values, and simple validation to make code more reliable.

## Key Takeaways

- Functions reduce repeated code.
- Parameters allow the same function to work with different datasets.
- `return` is more reusable than `print` for calculation logic.
- Empty-list checks prevent runtime errors.
- `None` is useful when no valid result exists.

## Improvement Notes

The first attempt correctly used functions for count, total, average, minimum, maximum, and range. The main improvement was adding empty-list handling and correcting the range function so it returns highest value minus lowest value.
